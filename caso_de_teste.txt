# Cenários de Teste para API RestFul-Booker

GET - Encontrar uma reserva pelo ID
- Deve retornar os dados da reserva correspondente.
- A resposta deve conter o status 200.

POST - Criar uma nova reserva
- Todos os campos obrigatórios devem ser preenchidos.
- A requisição deve possuir autorização.
- A resposta deve conter o status 200

PUT - Atualizar uma reserva
- Todos os campos obrigatórios devem ser preenchidos.
- A requisição deve possuir autorização.
- A resposta deve conter o status 200.

PATCH - Atualizar parcialmente uma reserva
- Devem ser enviados apenas campos válidos.
- A requisição deve possuir autorização.
- resposta deve conter o status 200.

DELETE - Deletar uma reserva
- Deve ser informado um ID válido.
- A requisição deve possuir autorização.
- A resposta deve conter o status 201
